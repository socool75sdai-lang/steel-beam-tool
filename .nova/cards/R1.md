## Defect found in QA + fix (deviation from spec)
The HANDOVER's prescribed onBlur (just re-firing `onChange` with `parseFloat`) does NOT clear a
leading zero in React 19. When the user types `010`, onChange already sets state to the number
`10`; onBlur re-firing `onChange({ span: 10 })` is a no-op value-wise, so React does not
reconcile the input's DOM text and `010` persists. Verified live: input displayed `010` while
the span-dependent `Le` readout correctly showed `10.00 m`.

**Fix:** onBlur now also imperatively normalizes the field:
```tsx
onBlur={(e) => {
  const v = parseFloat(e.target.value) || 0;
  onChange({ field: v });
  e.target.value = String(v);   // forces clean numeric display
}}
```
onChange is unchanged (must allow intermediate typing like `0.`).
